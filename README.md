# 42 Luxembourg – Common Core

This repository contains my project submissions for the **Common Core** curriculum at [42 Luxembourg](https://www.42luxembourg.lu) — a project-based, peer-evaluated engineering school with no teachers and no traditional classes.

Each project is written in C and built from scratch, following strict coding norms (the 42 Norm) and evaluated by peers.

---

## Projects

| Project | Description | Key concepts |
|---------|-------------|--------------|
| [libft](./libft) | Recreation of a subset of the C standard library | Memory management, linked lists, string manipulation |
| [ft_printf](./ft_printf) | Re-implementation of `printf` | Variadic functions, format parsing |
| [getnextline](./getnextline) | Function that reads a file line by line | File descriptors, static variables, buffers |
| [push_swap](./push_swap) | Sorting algorithm using two stacks with minimal operations | Algorithm design, complexity analysis |
| [so_long](./so_long) | Small 2D game using the MiniLibX graphics library | Game loop, event handling, 2D rendering |
| [minitalk](./minitalk) | Client-server communication using UNIX signals | Signals (SIGUSR1/SIGUSR2), bitwise operations |
| [philosophers](./philosophers) | Dining philosophers problem simulation | Threads, mutexes, race condition prevention |
| [minishell](./Minishell) | Simplified UNIX shell | Process management, pipes, redirections, parsing |

---

## About 42

42 is a tuition-free coding school that teaches through projects and peer review. There are no lectures — students learn by doing, reading documentation, and collaborating.

The Common Core is the foundational curriculum, covering systems programming in C, algorithms, Unix fundamentals, and collaborative development.
